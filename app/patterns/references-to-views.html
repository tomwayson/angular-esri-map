Getting a Reference to the Map and Scene Views

The map and scene view directives include function binding (events) as a way for you to get direct references to the underlying map and scene view objects from a parent controller. This will allow you to manipulate those objects in ways beyond what is allowed by the declarative directive API.

Events

The map and scene view directives each include both on-create and on-load events that pass a reference to the underlying map or scene view object to a callback function. The on-create event is fired immediately after the view has been created (i.e. new MapView()) and the on-load event fires after the view's promise has been resolved.

The map and scene view directives also include an on-error event that passes the error information object if a view's promise has been rejected. This can be useful, for example, if you wish to use a scene view directive but need to gracefully handle browsers that do not support WebGL. Most examples using the scene view directive also demonstrate how to use this binding.
